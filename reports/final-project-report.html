Executive Summary

This report documents the Cloud Attack Simulation Lab — a hands-on AWS security project that simulates real-world attacker behavior using industry-standard offensive security tooling. Three attack scenarios were executed against a live AWS account, covering reconnaissance, defense evasion, and data exfiltration techniques mapped to the MITRE ATT&CK framework.

The primary finding is that AWS GuardDuty with default configuration detected only 28% of attack actions. Seven specific detection gaps were identified with remediation recommendations for each. This project demonstrates attacker mindset, detection engineering depth, and the ability to communicate security findings at both technical and executive levels.

Detection Gap Analysis
Gap 1 — Severity Miscalibration
GuardDuty rates CloudTrail disabling as Low severity (2.0). In a real SOC this finding sits in a low priority queue and may not be investigated for hours — by which time the attacker has completed their objective.
Fix: EventBridge rule to auto-escalate Stealth:IAMUser/CloudTrailLoggingDisabled to Critical
Gap 2 — Alternative Disruption Methods
GuardDuty only detects the StopLogging API call. Modifying the S3 bucket policy to block log delivery, deleting the bucket, or modifying event selectors does not generate a Stealth finding.
Fix: AWS Config rule on CloudTrail log bucket policy changes
Gap 3 — Detection Lag Window
GuardDuty took approximately 10 minutes to generate a finding after StopLogging. A fast attacker can complete their objective before any alert fires.
Fix: CloudTrail → CloudWatch Logs → Lambda pipeline for sub-minute detection
Gap 4 — S3 Read Access Invisible
S3 GetObject calls using legitimate credentials generate zero GuardDuty findings by default. Complete CloudTrail log exfiltration was undetected.
Fix: Enable GuardDuty S3 Protection and AWS Macie on CloudTrail bucket
Gap 5 — CloudTrail Data Events Disabled
CloudTrail only logs management events by default. S3 object downloads do not appear in CloudTrail without explicit data event enablement.
Fix: Enable S3 data events on CloudTrail log bucket via put-event-selectors
Gap 6 — Silence Window Unrecoverable
Actions taken while CloudTrail is stopped leave no recoverable evidence. The gap is detectable but its contents are permanently lost.
Fix: VPC Flow Logs for network-level visibility independent of CloudTrail
Gap 7 — Finding Aggregation Reduces Urgency
GuardDuty increments counts on existing findings rather than creating new ones for repeated occurrences. SOC analysts treat aggregated findings with lower urgency.
Fix: EventBridge on every finding update for Stealth finding types
